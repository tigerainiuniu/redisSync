# 项目说明

本仓库为 **Redis 一对多同步服务** 的完整源码与本地联调环境（与 [GitHub redisSync](https://github.com/tigerainiuniu/redisSync) 根目录布局一致）。

## 目录结构

| 路径 | 说明 |
|------|------|
| `redis_sync/` | 同步核心：连接管理、全量 SCAN、增量检测、编排、Web UI 等 |
| `run_sync_service.py` | 服务入口（读取 YAML 配置并阻塞运行） |
| `config.yaml.example` | 配置模板；实际使用可复制为 `config.yaml`（已被 `.gitignore` 忽略） |
| `docker/redis-five/` | 本地 **1 源 + 4 目标** 共 5 个 Redis 容器的 Compose 与专用配置 |
| `scripts/e2e_five_redis.py` | 五实例冒烟：源写入 → 各目标全量 SCAN 迁移 → GET 校验 |
| `tests/` | pytest 单元/集成测试 |
| `reports/TEST_REPORT.md` | 测试报告（可随本地跑测更新） |

## 同步模式（配置项 `sync.mode`）

- **`hybrid`（推荐联调）**：先对每个目标执行 **全量** 迁移，再按间隔执行 **统一增量**（默认基于 `SCAN` + `OBJECT IDLETIME` 检测变更键，再 `DUMP`/`RESTORE` 到各目标）。
- **`full`**：仅全量，不进入增量循环。
- **`incremental`**：仅增量，跳过初始全量（适合源与目标已对齐的场景）。

详细字段说明见 `README.md` 与 `config.yaml.example`。

## 源 Redis 连接数（多目标时）

- 在 **迁移编排路径**上，所有目标共享 **同一个** 源端 `Redis` 客户端实例（首个目标 `connect_source`，其余 `set_source_client`），**不会**按目标数倍增源连接。
- 服务另维护 **`source_conn`**，用于 Web 与 **SYNC/PSYNC** 实时复制等，与编排器源客户端并存时，整体上为 **两份源客户端对象**（非「每目标一条」）。

## Docker 五实例联调

1. 启动容器：`cd docker/redis-five && docker compose up -d`
2. 宿主机端口：**源 `6378`**（避免与常见本机 `6379` 冲突），目标 **`6380`–`6383`**
3. 检查配置：  
   `python3 run_sync_service.py --config docker/redis-five/config.docker.yaml --check-config`
4. 冒烟脚本（在仓库根目录）：  
   `python3 scripts/e2e_five_redis.py`
5. 长驻同步（全量 + 增量 + Web，默认 `http://127.0.0.1:18080`）：  
   `python3 run_sync_service.py --config docker/redis-five/config.docker.yaml`

更短步骤见 `docker/redis-five/USAGE.txt`。
